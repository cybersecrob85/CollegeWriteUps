## Course Overview

This course covered the practical skills of building and managing virtual machine environments. Labs moved from creating and configuring individual VMs through more advanced operations like cloning, snapshots, file sharing between systems, and working with both Windows and Linux server platforms. The course also introduced Docker containerization as a final lab, directly connecting to the container skills used in later project work. Virtual computing is a foundational skill for any security or IT professional, as nearly every modern lab, test environment, and cloud deployment relies on virtualization.

---

## What's Covered

**VM Creation and Configuration** — Built and configured Ubuntu and Windows virtual machines from scratch, including OS installation, updates, and initial system setup. Covered best practices for naming, resource allocation, and network settings.

**Clones, Snapshots, and VM Management** — Explored the difference between full and linked clones, how to create and restore snapshots for safe testing, and how to manage VM states effectively. Snapshots are a critical tool in any security lab environment for rolling back after testing malware or configurations.

**File Sharing Between VMs** — Configured shared folders and file sharing between virtual machines and the host system, covering both VMware and cross-platform sharing scenarios.

**System Maintenance** — Covered routine VM maintenance tasks including updates, storage management, and keeping virtual environments healthy and performant.

**Windows 11 Pro and Windows Server 2025** — Deployed and configured Windows 11 Pro and Windows Server 2025 virtual machines alongside Rocky Linux, practicing multi-OS environment management.

**Docker Basics** — Final lab introduced Docker containerization, covering pulling images, running containers, and basic container management — skills that connected directly into the Project and Portfolio II Docker deployment.
